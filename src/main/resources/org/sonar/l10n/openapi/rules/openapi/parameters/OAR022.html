In "get" operations, the $orderby parameter, which allows to indicate the order, must be defined as a non-required parameter.

Noncompliant Code Example (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $orderby
          required: false
          type: array
          items:
            type: string
      responses:
        200:
          description: Ok
