Normative - API Definition

Allowing any response headers to be set and not specifying the required response headers can cause some vulnerabilities in the API

Noncompliant Code Example (OpenAPI 2)

All the examples are in YAML

swagger: "2.0"
info:
    version: 1.0.0
    title: Swagger Petstore
paths:
    /pets:
    get:
        responses:
        '200':
            description: OK

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
    version: 1.0.0
    title: Swagger Petstore
paths:
    /pets:
    get:
        responses:
        '200':
            description: OK
            headers:
            strinct-transport-security:
                type: string
            x-frame-options:
                type: string
            referrer-policy:
                type: string
            x-content-type-options:
                type: string
            x-xss-protection:
                type: string

Noncompliant Code Example (OpenAPI 3)

openapi: "3.0.0"
info:
    version: 1.0.0
    title: Swagger Petstore
components:
    responses:
    PetsResponse:
        description: OK    
paths:
    /pets:
    get:
        responses:
        '200':
            $ref: '#/components/responses/PetsResponse'

Compliant Solution (OpenAPI 3)

openapi: "3.0.0"
info:
    version: 1.0.0
    title: Swagger Petstore
components:
    responses:
    PetsResponse:
        description: OK
        headers:
        strinct-transport-security:
            schema:
            type: string
        x-frame-options:
            schema:
            type: string
        referrer-policy:
            schema:
            type: string
        x-content-type-options:
            schema:
            type: string
        x-xss-protection:
            schema:
            type: string     
paths:
    /pets:
    get:
        responses:
        '200':
            $ref: '#/components/responses/PetsResponse'
