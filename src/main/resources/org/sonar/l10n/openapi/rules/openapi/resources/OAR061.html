Response codes in GET operations must be defined according to the standard

Noncompliant Code Example (OpenAPI 2)

All the examples are in YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        400:
          description: Bad request

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: OK

Noncompliant Code Example (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        400:
          description: Bad request

Compliant Solution (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
    200:
