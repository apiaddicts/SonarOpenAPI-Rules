In operations with paged response, the $total parameter, which is used to indicate whether the total of existing records is to be returned, must be defined as a non-required parameter.

Noncompliant Code Example (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $total
          required: false
          type: boolean
          default: false
      responses:
        200:
          description: Ok

Noncompliant Code Example (OpenAPI 3)

openapi: 3.0.1
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 3)

openapi: 3.0.1
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $total
          required: false
          schema:
            type: boolean
            default: false
      responses:
        200:
          description: Ok
