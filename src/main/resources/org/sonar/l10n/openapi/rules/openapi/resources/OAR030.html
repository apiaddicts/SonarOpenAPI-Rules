An API without a status endpoint is hard to observe with monitoring tools.

Noncompliant Code Example (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok
  /status:
    get:
      responses:
        200:
          description: Ok
