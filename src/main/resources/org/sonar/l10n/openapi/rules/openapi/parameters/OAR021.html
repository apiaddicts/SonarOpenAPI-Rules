In "get" operations, the $exclude parameter, which allows to indicate the attributes that will not be returned in the response, must be defined as a non-required parameter.

Noncompliant Code Example (OpenAPI 2)

All the examples are in YAML

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $exclude
          required: false
          type: array
          items:
            type: string
      responses:
        200:
          description: Ok
