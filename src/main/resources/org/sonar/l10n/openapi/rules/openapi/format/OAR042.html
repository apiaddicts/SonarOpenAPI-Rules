Base path must must be compliant with the standard

The first part of base path corresponds to the name of the API, starting with the prefix "api-" and must have at least one character: /api-xxxx

The second part of base path corresponds to the version of the API and must start with the prefix "v" continuing with the version number in integer format: /v1

Noncompliant Code Example (OpenAPI 2)

JSON

{
    "swagger": "2.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "basePath": "/seguros", 
    "paths": {}
}

YAML

swagger: "2.0"
info:
    version: 1.0.0
    title: Swagger Petstore
basePath: /seguros
paths: {}

Compliant Solution (OpenAPI 2)

JSON

{
    "swagger": "2.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "basePath": "/api-seguros/v1", 
    "paths": {}
}

YAML

swagger: "2.0"
info:
    version: 1.0.0
    title: Swagger Petstore
basePath: /api-seguros/v1
paths: {}

Noncompliant Code Example (OpenAPI 3)

JSON

{
    "openapi": "3.0.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "servers": [
        {
            "url": "https://petstore.swagger.io/seguros/version/1"
        }
    ], 
    "paths": {}
}

YAML

openapi: "3.0.0"
info:
    version: 1.0.0
    title: Swagger Petstore
servers:
    - url: https://petstore.swagger.io/seguros/version/1
paths: {}

Compliant Solution (OpenAPI 3)

JSON

{
    "openapi": "3.0.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "servers": [
        {
            "url": "https://petstore.swagger.io/api-seguros/v1"
        }
    ], 
    "paths": {}
}

YAML

openapi: "3.0.0"
info:
    version: 1.0.0
    title: Swagger Petstore
servers:
    - url: https://petstore.swagger.io/api-seguros/v1
paths: {}
