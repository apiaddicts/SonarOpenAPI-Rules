Normative - API Definition

  • STANDARD: API Definition - Nomenclature

OpenAPI 2: Base path must be compliant with the standard.

OpenAPI 3: The path of 'server' url must be compliant with the standard.

Noncompliant Code Example (OpenAPI 2)

swagger: "2.0"
info:
    version: 1.0.0
    title: Swagger Petstore
basePath: /seguros
paths: {}

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
    version: 1.0.0
    title: Swagger Petstore
basePath: /api-seguros/v1
paths: {}

Noncompliant Code Example (OpenAPI 3)

openapi: "3.0.0"
info:
    version: 1.0.0
    title: Swagger Petstore
servers:
    - url: https://petstore.swagger.io/seguros/version/1
paths: {}

Compliant Solution (OpenAPI 3)

openapi: "3.0.0"
info:
    version: 1.0.0
    title: Swagger Petstore
servers:
    - url: https://petstore.swagger.io/api-seguros/v1
paths: {}
