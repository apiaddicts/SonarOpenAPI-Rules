Se debe de definir una convención de nombres para los parámetros de ruta, los parámetros de consulta, las propiedades del cuerpo de la solicitud y las propiedades del cuerpo de la respuesta.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets/{pet-id}:
    patch:
      consumes:
        - application/json
      produces:
        - application/json
      parameters:
        - in: path
          name: pet-id
          type: integer
          required: true
        - in: query
          name: pet_Type
          type: string
        - in: body
          name: pet
          description: Pet
          schema:
            type: object
            required:
              - PET_NAME
            properties:
              PET_NAME:
                type: string
              photoUrls:
                type: array
                items:
                  type: string
      responses:
        200:
          description: OK
          schema:
            type: object
            properties:
              petId:
                type: integer
              petType:
                type: string
              petName:
                type: string
              photoUrls:
                type: array
                items:
                  type: string
