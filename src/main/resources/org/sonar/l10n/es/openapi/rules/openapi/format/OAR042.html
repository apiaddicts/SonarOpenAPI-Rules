La ruta base debe cumplir con el estándar.

La primera parte de la ruta base corresponde al nombre de la API, iniciando con el prefijo “api-” y debe tener al menos un carácter: /api-xxxx

La segunda parte de la ruta base corresponde a la versión de la API y debe iniciar con el prefijo “v” continuando con el número de versión de tipo entero: /v1

Ejemplo de código no compatible (OpenAPI 2)

JSON

{
    "swagger": "2.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "basePath": "/seguros", 
    "paths": {}
}

YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
basePath: /seguros
paths: {}

Solución compatible (OpenAPI 2)

JSON

{
    "swagger": "2.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "basePath": "/api-seguros/v1", 
    "paths": {}
}

YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
basePath: /api-seguros/v1
paths: {}

Ejemplo de código no compatible (OpenAPI 3)

JSON

{
    "openapi": "3.0.0", 
    "info": {
        "version": "1.0.0", 
        "title": "Swagger Petstore"
    }, 
    "servers": [
        {
            "url": "https://petstore.swagger.io/seguros/version/1"
        }
    ], 
    "paths": {}
}

YAML

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
servers:
  - url: https://petstore.swagger.io/seguros/version/1
paths: {}
