Normativa - Definición de APIs

  • NORMA: Definición API - Nomenclatura

OpenAPI 2: El valor de 'basePath' debe cumplir con el estándar.

OpenAPI 3: El ruta de la URL de 'server' debe cumplir con el estándar.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
basePath: /seguros
paths: {}

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
basePath: /api-seguros/v1
paths: {}

Ejemplo de código no compatible (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
servers:
  - url: https://petstore.swagger.io/seguros/version/1
paths: {}

Solución compatible (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
servers:
  - url: https://petstore.swagger.io/api-seguros/v1
paths: {}
