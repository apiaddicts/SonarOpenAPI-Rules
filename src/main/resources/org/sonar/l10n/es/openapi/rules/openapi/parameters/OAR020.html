En operaciones de consulta, el parámetro $expand, que permite expandir un sub-elemento, se debe definir como parámetro no obligatorio.

Ejemplo de código no compatible (OpenAPI 2)

Todos los ejemplos están en YAML

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $expand
          type: array
          required: false
          items:
            type: string
      responses:
        200:
          description: Ok
