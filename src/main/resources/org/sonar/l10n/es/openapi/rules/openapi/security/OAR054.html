Normativa - Definición de APIs

  • NORMA: Definición API - Nomenclatura

El nombre de host debe ser un subdominio del nombre de dominio de la organización.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
host: other.domain.com
paths: {}

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
host: api.cloudappi.net
paths: {}

Ejemplo de código no compatible (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
servers:
  - url: https://aws-dev-cloudappi.net/api-vehicular/v1/
  - url: https://aws-test.cloudappi.net/api-vehicular/v1/
  - url: https://aws-cloudappi.net/api-vehicular/v1/
paths: {}

Solución compatible (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
servers:
  - url: https://api-dev.cloudappi.net/api-vehicular/v1/
  - url: https://api-test.cloudappi.net/api-vehicular/v1/
  - url: https://api.cloudappi.net/api-vehicular/v1/
paths: {}
