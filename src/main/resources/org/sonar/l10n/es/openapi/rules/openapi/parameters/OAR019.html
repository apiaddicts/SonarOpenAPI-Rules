En operaciones de consulta, el parámetro $select, que permite indicar los atributos a devolver en la respuesta, se debe definir como parámetro no obligatorio.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $select
          required: false
          type: array
          items:
            type: string
      responses:
        200:
          description: Ok

Ejemplo de código no compatible (OpenAPI 3)

openapi: 3.0.1
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 3)

openapi: 3.0.1
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
      - name: $select
        in: query
        required: false
        schema:
          type: array
          items:
            type: string
      responses:
        200:
          description: Ok
